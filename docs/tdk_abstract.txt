Large language models are trained on fixed snapshots of public knowledge. Libraries released after the snapshot are missing from the model's parametric knowledge and therefore from the code it recommends and completes. This paper estimates the causal effect of that absence on the subsequent diffusion of newly released Python libraries.

I exploit the September 2021 training cutoff shared by the GPT-3.5 and GPT-4 model families. The running variable is library release week relative to the cutoff, constructed from first observed PyPI download activity. The main estimator is a Difference-in-Discontinuities (Diff-in-RDD) design that nets the 2021 regression discontinuity against placebo discontinuities at the same calendar week in 2018, 2019, and 2020, which removes seasonal release patterns. Estimation uses weighted local linear regression with a 13-week bandwidth, an asymmetric 9-week donut that drops late-summer releases with ambiguous training inclusion, and standard errors clustered at the year-by-week level. The sample covers 527,361 PyPI packages and a 28,243-library GitHub-matched subsample.

For libraries that reached 500 downloads in their first 26 weeks (a quality threshold that closes in March 2022, entirely before ChatGPT), the baseline-adjusted post-AI download discontinuity is -7.3 log points (p = 0.002). The same post-cutoff group was 28.6 percentage points more likely than pre-cutoff libraries to clear that threshold in the pre-AI period, so the post-AI reversal cannot be attributed to a quality difference between groups. The discontinuity in GitHub code imports (-26.8) is roughly twice the matching PyPI estimate (-13.1), pointing to LLM-mediated code generation rather than general discoverability as the channel. None of the placebo years produces a comparable jump. The gap is dormant before the November 2022 ChatGPT launch, opens only afterward, and is still widening as of January 2026: pre-cutoff libraries average roughly five times the weekly downloads of post-cutoff libraries, against a modest pre-ChatGPT baseline of about 28 percent.

The estimate is local to one cutoff in one ecosystem and should not be read as a general claim about LLMs and software diffusion. Within those bounds, it shows that the temporal boundary of training data shapes which newly released tools developers actually adopt.
