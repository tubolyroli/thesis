Large language models (LLMs) are trained on fixed snapshots of public knowledge, creating a hard temporal boundary between tools the model can recommend and tools it cannot. This paper asks whether that boundary shapes the early diffusion of newly released open-source Python libraries.

Research question: Do Python libraries released shortly before a major LLM knowledge cutoff diffuse more widely than libraries released shortly after, because the former are represented in the model's pretrained knowledge?

The identification strategy exploits the documented September 2021 training cutoff of the GPT-3.5 and GPT-4 model families. The running variable is library release timing in weeks relative to the cutoff, constructed from first observed PyPI download activity. The primary estimator is a Difference-in-Discontinuities (Diff-in-RDD) design that differences the 2021 regression discontinuity against placebo discontinuities estimated at the same calendar position in 2018-2020, removing seasonal confounds. Inference uses bias-corrected local linear regression with data-driven bandwidths and cluster-robust standard errors. The sample covers 527,361 PyPI packages and 28,243 GitHub-matched libraries.

Three results emerge. The Diff-in-RDD estimate for libraries that achieved at least 500 downloads in their first 26 weeks (measured entirely before ChatGPT) is -12.8 log points (p = 0.008), indicating a significant post-AI diffusion penalty for post-cutoff entrants. Post-cutoff libraries were 14.5 percentage points more likely to reach this quality threshold before ChatGPT, making the reversal conservative. The penalty is approximately twice as large in GitHub code imports (26.8 log points) as in PyPI downloads (13.1 log points), consistent with LLM-steered code generation as the operative channel. The gap was absent in all three placebo years and emerged only after November 2022; no catch-up is observed through January 2026.

The findings provide quasi-experimental evidence that LLM training cutoffs function as a structural barrier to entry for newly released software, redirecting adoption toward tools already present in training data. This Knowledge Wall operates broadly across the ecosystem rather than only in identifiably AI-heavy contexts, and it has persisted for over three years without correction.
